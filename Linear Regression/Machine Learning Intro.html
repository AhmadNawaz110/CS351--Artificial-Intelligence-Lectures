Cross-Validation

Robust model evaluation technique

What is Cross-Validation?

A technique to assess model performance by splitting data into multiple subsets and training/evaluating multiple times

K-Fold Cross-Validation

Types of Cross-Validation

K-Fold

Data split into K equal parts, each used once as test set

Stratified K-Fold

Preserves class distribution in each fold

Leave-One-Out

Each sample is used once as test set (K = N)

Time Series

Chronological splits for temporal data

Model Evaluation Metrics

Measuring ML model performance

Classification Metrics

Accuracy

(TP + TN) / (TP + TN + FP + FN)

Overall correctness

Precision

TP / (TP + FP)

False positive avoidance

Recall

TP / (TP + FN)

Sensitivity, true positive rate

F1-Score

2 × (Precision × Recall) / (Precision + Recall)

Balance between precision and recall

Regression Metrics

MAE

Mean Absolute Error

Average absolute difference

RMSE

Root Mean Square Error

Penalizes larger errors

Overfitting and Underfitting

Finding the right model complexity

Common Model Issues

Overfitting

Model learns training data too well

High training accuracy

Low test accuracy

Fails to generalize

Underfitting

Model is too simple to capture patterns

Low training accuracy

Low test accuracy

Fails to learn patterns

The Bias-Variance Tradeoff

Solutions

For Overfitting:

More training data, regularization, feature selection, simpler model

For Underfitting:

More features, complex model, longer training, feature engineering

Source: Andrew Ng's Machine Learning Course, 2023

Challenges with Cybersecurity Datasets

Overcoming obstacles in security ML

Key Challenges

Class Imbalance

Attack samples are rare compared to normal traffic

Models biased toward majority class

Solutions: Resampling, cost-sensitive learning

Privacy Concerns

Real data may contain sensitive information

Limited availability of real datasets

Solutions: Anonymization, synthetic data

Evolving Threats

Attack patterns change rapidly

Models become outdated quickly

Solutions: Continuous learning, ensemble methods

Data Quality

Incomplete or mislabeled data

Noisy labels affect model performance

Solutions: Label cleaning, active learning

Best Practices

Ethical Considerations in ML for Security

Responsible use of AI in cybersecurity

Key Ethical Challenges

Privacy

ML systems may process sensitive data

Potential for surveillance overreach

Solutions: Data minimization, anonymization

Bias

Models may inherit or amplify biases

Unfair targeting of certain groups

Solutions: Diverse training data, bias audits

Accountability

Who is responsible for ML decisions?

Black box nature of complex models

Solutions: Explainable AI, human oversight

Dual Use

Security ML can be weaponized

Adversarial attacks on ML systems

Solutions: Robustness testing, security reviews
